# Calling External APIs from Apex

### 1. Overview

Apex provides built-in classes to perform HTTP callouts to external systems. These are primarily used in integrations where Salesforce communicates with external APIs.

```mermaid
flowchart TD
    A[Apex Code] --> B[HttpRequest]
    B --> C[HTTP.send()]
    C --> D[HttpResponse]
    D --> E[Process Response]
```

---

# 2. Core Classes for Callouts

## 2.1 HttpRequest

### Description

`HttpRequest` is used to construct and configure the outbound HTTP request.

### Common Methods

| Method                      | Description                                                   | When to Use                 |
| --------------------------- | ------------------------------------------------------------- | --------------------------- |
| `setEndpoint(String)`       | Sets the target API URL                                       | Always required             |
| `setMethod(String)`         | Defines HTTP method (`GET`, `POST`, `PUT`, `DELETE`, `PATCH`) | Based on API operation      |
| `setHeader(String, String)` | Adds HTTP headers                                             | For auth, content type      |
| `setBody(String)`           | Sets request payload                                          | Required for POST/PUT/PATCH |
| `setTimeout(Integer)`       | Sets timeout in milliseconds                                  | For long-running APIs       |

### Example Usage

```apex
HttpRequest req = new HttpRequest();
req.setEndpoint('https://api.example.com/data');
req.setMethod('GET');
req.setHeader('Content-Type', 'application/json');
```

---

## 2.2 HTTP

### Description

`HTTP` class is responsible for sending the request and receiving the response.

### Key Method

| Method                  | Description                             | When to Use                    |
| ----------------------- | --------------------------------------- | ------------------------------ |
| `send(HttpRequest req)` | Sends HTTP request and returns response | Always used to execute callout |

### Example Usage

```apex
Http http = new Http();
HttpResponse res = http.send(req);
```

---

## 2.3 HttpResponse

### Description

`HttpResponse` holds the response returned from the external API.

### Common Methods

| Method              | Description                               | When to Use           |
| ------------------- | ----------------------------------------- | --------------------- |
| `getStatusCode()`   | Returns HTTP status code (200, 400, etc.) | Check success/failure |
| `getStatus()`       | Returns status message                    | Debugging             |
| `getBody()`         | Returns response body                     | Process data          |
| `getHeader(String)` | Retrieves specific header                 | Advanced use          |

### Example Usage

```apex
Integer statusCode = res.getStatusCode();
String responseBody = res.getBody();
```

---

# 3. When to Use Which Class

```mermaid
flowchart TD
    A[Start Callout] --> B[Create HttpRequest]
    B --> C[Configure Request]
    C --> D[Use HTTP.send()]
    D --> E[Get HttpResponse]
    E --> F[Handle Response]
```

- Use **HttpRequest** → when preparing request details
- Use **HTTP** → when sending request
- Use **HttpResponse** → when handling response

# 4. Example: Apex Callout with Exception Handling

## Scenario

Send a GET request to an external API and handle possible errors safely.

```java
public class ExternalAPICallout {

    public static void fetchData() {

        HttpRequest req = new HttpRequest();
        req.setEndpoint('https://api.example.com/data');
        req.setMethod('GET');
        req.setHeader('Content-Type', 'application/json');
        req.setTimeout(60000);

        Http http = new Http();

        try {
            HttpResponse res = http.send(req);

            // Check status code
            if (res.getStatusCode() == 200) {
                String responseBody = res.getBody();
                System.debug('Success Response: ' + responseBody);

                // You can deserialize JSON here if needed
            } else {
                System.debug('Error Status Code: ' + res.getStatusCode());
                System.debug('Error Response: ' + res.getBody());
            }

        } catch (System.CalloutException e) {
            System.debug('Callout Exception: ' + e.getMessage());

        } catch (Exception e) {
            System.debug('General Exception: ' + e.getMessage());
        }
    }
}
```

# 5. Important Considerations

## 5.1 Remote Site Settings / Named Credentials

- Endpoint must be whitelisted using:
  - Remote Site Settings, or
  - Named Credentials (recommended)

## 5.2 Governor Limits

- Maximum **100 callouts per transaction**

## 5.3 Asynchronous Callouts

Use when:

- Long-running requests
- Large data processing

Supported via:

- `@future(callout=true)`
- `Queueable Apex`
- `Batch Apex`

---

# 6. Best Practices

- Use **Named Credentials** for authentication
- Always implement **exception handling**
- Validate **status codes before processing**
- Avoid hardcoding endpoints
- Use **JSON.deserialize()** for structured responses
- Keep callouts in **service classes**

---

# 7. Summary

- `HttpRequest` → prepares request
- `HTTP` → sends request
- `HttpResponse` → handles response
- Always handle exceptions and validate responses
- Prefer Named Credentials for secure integrations
